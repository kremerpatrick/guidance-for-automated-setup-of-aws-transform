# Guidance for Automated Setup of AWS Transform

## Overview

This guidance provides automated Infrastructure as Code (IaC) solutions for deploying AWS Transform resources to modernize and migrate your workloads to AWS. Choose the appropriate guide below based on your specific workload type.

### Benefits

**Simplified Setup**: Infrastructure as Code automates AWS Transform deployment and configuration, reducing setup time and manual steps.

**Secure Foundation**: Implements AWS security best practices through IAM Identity Center configuration and role-based access control.

**Guided Approach**: Provides step-by-step instructions and templates for configuring AWS Transform and associated services.

## VMware Workloads

For organizations modernizing VMware workloads, this [guide](READMEvmware.md) walks you through the complete journey from discovery and assessment of your on-premises VMware environment to migration execution. This Guidance provides an automated approach to deploy AWS Transform for VMware resources using infrastructure as code (IaC), streamlining the setup process by automating the provisioning of required AWS services, network configurations, and security controls. The guidance accelerates time-to-value for organizations migrating and modernizing their VMware workloads while ensuring adherence to AWS best practices and security standards.

## .NET Workloads

For organizations modernizing .NET applications, this [guide](READMEdotnet.md) helps you assess, plan, and execute your migration to AWS. This Guidance delivers an automated infrastructure as code (IaC) solution to deploy AWS Transform for .NET, eliminating manual configuration steps and reducing deployment complexity. By automating the provisioning of essential AWS services, IAM Identity Center setup, and security configurations, this guidance enables your teams to quickly establish a secure foundation for analyzing .NET workloads, identifying modernization opportunities, and accelerating your cloud transformation while maintaining compliance with AWS security best practices.
